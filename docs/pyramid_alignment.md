# 金字塔配准详细说明

## 📊 配准架构对比

### 原始方案：单层配准

```
输入特征 (32×H×W)
    ↓
[3 层卷积]
    ↓
光流场 (2×H×W)
```

**问题**:
- 只能估计小范围位移
- 对大位移不敏感
- 容易陷入局部最优

---

### v2.1+ 方案：双层金字塔配准

```
输入特征 (32×H×W)
    ↓
    ├──────────────────┐
    ↓                  ↓
[下采样 0.5x]      [原始分辨率]
    ↓                  ↓
[粗配准网络]      [精配准网络]
    ↓                  ↓
flow_coarse       flow_fine
(2×H×W)          (2×H×W)
    ↓                  ↓
    └───── 叠加 ───────┘
           ↓
    flow = flow_coarse + flow_fine
```

---

## 🔧 实现细节

### 1. 粗配准层 (Coarse Alignment)

```python
# 下采样特征 (H/2 × W/2)
concat_low = F.interpolate(concat_for_flow, scale_factor=0.5)

# 粗配准网络
flow_coarse = self.aligner_coarse(concat_low)  # (2×H/2×W/2)

# 上采样回原始分辨率，并放大光流 (位移也放大 2 倍)
flow_coarse = F.interpolate(flow_coarse, scale_factor=2.0) * 2.0
```

**作用**:
- 在下采样特征上估计**大范围位移**
- 感受野更大，能捕捉全局运动
- 计算量小，速度快

**优势**:
- ✅ 能处理大位移 (±10 像素以上)
- ✅ 对噪声不敏感
- ✅ 提供粗略但全局的对齐

---

### 2. 精配准层 (Fine Alignment)

```python
# 在原始分辨率上估计小位移
flow_fine = self.aligner_fine(concat_for_flow)  # (2×H×W)
```

**作用**:
- 在原始分辨率上估计**小范围位移**
- 捕捉局部细节运动
- 修正粗配准的误差

**优势**:
- ✅ 能处理精细的局部形变
- ✅ 保留高频细节
- ✅ 修正粗配准的残差

---

### 3. 光流叠加

```python
# 最终光流 = 粗配准 + 精配准
flow = flow_coarse + flow_fine
```

**原理**:
- 粗配准提供**全局位移场**
- 精配准提供**局部残差修正**
- 两者叠加得到**完整位移场**

## 🎯 适用场景

### ✅ 适合使用金字塔配准的场景

1. **摄像头轻微晃动**
   - 手持拍摄产生的抖动
   - 风吹导致的摄像头摆动

2. **目标移动**
   - 行人在两帧之间移动
   - 车辆行驶产生的视角变化

3. **光照变化导致的表观运动**
   - 云层遮挡导致的阴影移动
   - 车灯照射角度变化

### ❌ 不适合的场景

1. **剧烈运动模糊**
   - 快速移动导致的模糊
   - 配准无法恢复模糊信息

2. **遮挡区域**
   - 新出现的区域无法配准
   - 需要特殊处理 (如掩码)

## 🎓 技术背景

### 为什么使用金字塔结构？

金字塔结构在光流估计中广泛应用，原因:

1. **多尺度分析**
   - 粗尺度：捕捉大位移，全局运动
   - 细尺度：捕捉小位移，局部细节

2. **由粗到精优化**
   - 避免陷入局部最优
   - 提高配准精度

3. **计算效率**
   - 粗尺度计算量小
   - 细尺度只处理残差

### 相关论文

- **FlowNet 2.0** (CVPR 2017): 级联光流估计
- **PWC-Net** (CVPR 2018): 金字塔光流网络
- **LiteFlowNet** (CVPR 2018): 轻量级光流
